HTTP-API-Steuerung (Anleitung)

Der Loxone Miniserver oder andere Smart-Home-Komponenten können alle Sonos-Lautsprecher über einfache HTTP-GET-Anfragen steuern. Die Basis-URL lautet:

http://127.0.0.1:8888
GET /:raum/play

Startet die Musikwiedergabe im angegebenen Raum.

Beispiel: http://127.0.0.1:8888/wohnzimmer/play
GET /:raum/pause

Pausiert die Musikwiedergabe im angegebenen Raum.

Beispiel: http://127.0.0.1:8888/wohnzimmer/pause
GET /:raum/volume/:wert

Regelt die Lautstärke im angegebenen Raum. Unterstützt absolute Werte (0 bis 100) sowie relative Anpassungen mit + oder - Präfix.

Beispiele:
  • Lautstärke auf 25% setzen: http://127.0.0.1:8888/wohnzimmer/volume/25
  • Lautstärke um 5% erhöhen: http://127.0.0.1:8888/wohnzimmer/volume/+5
  • Lautstärke um 10% verringern: http://127.0.0.1:8888/wohnzimmer/volume/-10
GET /:raum/favorite/:name/:volume?

Spielt einen in der Sonos-App hinterlegten Favoriten (z. B. einen Radio-Sender) ab. Die Lautstärke ist optional. Der Name wird case-insensitiv verglichen.

Beispiele:
  • Favorit abspielen: http://127.0.0.1:8888/wohnzimmer/favorite/Radio%20Superfly
  • Favorit mit 25% Lautstärke abspielen: http://127.0.0.1:8888/wohnzimmer/favorite/Radio%20Superfly/25
GET /:raum/say/:text/:volume?

Führt eine Sprachansage (TTS) im Raum aus. Die Lautstärke ist optional. Nach Ende der Durchsage wird der vorherige Wiedergabezustand automatisch wiederhergestellt.

Beispiel (mit Lautstärke 40%): http://127.0.0.1:8888/wohnzimmer/say/Die%20Waschmaschine%20ist%20fertig/40
GET /:raum/tunein/play/:stationId

Spielt einen TuneIn-Radiosender über seine Stations-ID ab.

Beispiel: http://127.0.0.1:8888/kueche/tunein/play/68225
GET /:raum/leave

Trennt einen Lautsprecher aus einer bestehenden Gruppe, so dass er wieder eigenständig betrieben wird.

Beispiel: http://127.0.0.1:8888/kueche/leave
GET /:raum/clip/:file/:volume?

Spielt eine Sounddatei (z. B. bell.mp3) aus dem Ordner static/clips ab. Die Lautstärke ist optional.

Beispiel: http://127.0.0.1:8888/wohnzimmer/clip/bell.mp3/50
GET /preset/:name

Aktiviert eine vordefinierte Raumgruppe (Preset) aus dem Verzeichnis presets (z. B. Raumkombinationen, Lautstärken, Modi).

Beispiel: http://127.0.0.1:8888/preset/all

Priorisierte Befehlswarteschlange (Priority Queue)

Um UPnP-Konflikte bei schnell aufeinanderfolgenden Befehlen (z. B. mehreren TTS-Sprachansagen) zu vermeiden, verarbeitet die Bridge alle Aktionen für jeden Raum nacheinander (sequentiell) über eine Warteschlange.

GET ...?prio=true

Durch Anhängen des Parameters ?prio=true oder ?prio=1 an eine beliebige HTTP-GET-URL wird der Befehl an die erste Stelle der Warteschlange gesetzt (wird als nächstes ausgeführt).

Beispiel: http://127.0.0.1:8888/wohnzimmer/say/Wichtige%20Meldung/40?prio=true
GET / POST Sofortiger Abbruch bei Pause

Ein pause-Befehl bricht die Warteschlange für den Raum sofort ab, löscht alle ausstehenden Befehle und pausiert den Lautsprecher unverzüglich.

POST /api/control

Bei Steuerung über die POST-Schnittstelle kann die Priorisierung durch das Hinzufügen von "priority": true oder "prio": true im JSON-Payload angefordert werden.

{
  "room": "wohnzimmer",
  "action": "say",
  "value": "Wichtige Meldung",
  "priority": true
}
